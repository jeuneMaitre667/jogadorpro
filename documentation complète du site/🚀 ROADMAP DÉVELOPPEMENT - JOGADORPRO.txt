🚀 ROADMAP DÉVELOPPEMENT - JOGADORPRO
📋 CONTEXTE & OBJECTIFS
Projet : Plateforme de skill challenges pour paris sportifs (prop firm model)
Marché : Brésil
Lancement MVP :  Mai 2026
Lancement Marketing : Mai,Juin 2026 (début Coupe du Monde)
Temps disponible : 12 semaines (1 Fev - 1er Mai)
Business Model :
* Utilisateurs paient pour challenges (49€ ou 139€)
* Passent 2 phases avec règles strictes (25% puis 30% profit)
* S'ils réussissent → compte "funded" (80/20 split)
* On paie leurs profits bi-mensuels via PIX
* On se rémunère sur les échecs (95%+ des participants)

Sprint 3-4 : Challenge System (Semaines 3-4)
Modèle Challenge
* [ ] Table user_challenges (voir schema)
* [ ] 2 tiers : 1K (49€) et 5K (139€)
* [ ] 2 phases : Phase 1 (objectif 25%) / Phase 2 (objectif 30%)
* [ ] Timer 31 jours par phase
* [ ] Status : active, passed, failed, funded
Business Logic - Règles Challenge
* [ ] Stake : 1% min, 5% max du balance
* [ ] DD journalier : max 5% depuis balance début journée
* [ ] DD total : max 10% depuis capital initial
* [ ] Plafond gains journalier : 8% max
* [ ] Picks minimum : 20 par phase
* [ ] Jours actifs minimum : 15/31 jours (au moins 1 pick/jour)
Calculs Automatiques
* [ ] Balance tracking temps réel
* [ ] Profit percentage calculation
* [ ] DD calculation (journalier + total)
* [ ] Compteur picks
* [ ] Compteur jours actifs
* [ ] Vérification rules à chaque pick
Auto-Fail Triggers
* [ ] DD journalier > 5% → fail immédiat
* [ ] DD total > 10% → fail immédiat
* [ ] Fin des 31 jours sans objectif atteint → fail
* [ ] Violation stake (< 1% ou > 5%) → pick refusé
Phase Completion
* [ ] Vérif : profit >= 25% (phase 1) ou 30% (phase 2)
* [ ] Vérif : picks >= 20
* [ ] Vérif : jours actifs >= 15
* [ ] Si OK → status = 'passed'
* [ ] Si Phase 1 passed → créer Phase 2 automatiquement (délai 3j)
* [ ] Si Phase 2 passed → créer funded account
Challenge Démo Gratuit
* [ ] Capital : 100€ fictif
* [ ] Durée : 7 jours
* [ ] Objectif : 10% profit
* [ ] Picks min : 5
* [ ] DD max : 15% (plus permissif)
* [ ] Génération code promo automatique si succès (-30%)

🗄️ DATABASE SCHEMA COMPLET
-- USERS
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  name VARCHAR(100),
  country VARCHAR(2) DEFAULT 'BR',
  kyc_verified BOOLEAN DEFAULT FALSE,
  kyc_status VARCHAR(20), -- pending, verified, rejected
  fraud_risk_score INTEGER DEFAULT 0,
  flagged_for_review BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);


-- USER CHALLENGES
CREATE TABLE user_challenges (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  challenge_type VARCHAR(20) NOT NULL, -- 'demo_100', '1k', '5k'
  phase INTEGER NOT NULL, -- 1 ou 2
  status VARCHAR(20) DEFAULT 'active', -- active, passed, failed, funded
  
  -- Balance & Tracking
  initial_balance DECIMAL(10,2) NOT NULL,
  current_balance DECIMAL(10,2) NOT NULL,
  max_balance_reached DECIMAL(10,2),
  
  -- Rules Tracking
  picks_count INTEGER DEFAULT 0,
  active_days_count INTEGER DEFAULT 0,
  daily_dd_current DECIMAL(5,2) DEFAULT 0,
  total_dd_current DECIMAL(5,2) DEFAULT 0,
  
  -- Dates
  start_date TIMESTAMP NOT NULL,
  end_date TIMESTAMP NOT NULL,
  completed_at TIMESTAMP,
  
  -- Fail Reason
  fail_reason VARCHAR(50), -- DD_DAILY_EXCEEDED, DD_TOTAL_EXCEEDED, TIME_EXPIRED, etc.
  fail_dd_value DECIMAL(5,2),
  
  -- Add-ons
  success_refund_enabled BOOLEAN DEFAULT FALSE,
  insurance_level VARCHAR(20), -- null, silver, gold, platinum
  insurance_resets_remaining INTEGER DEFAULT 0,
  
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);


-- PICKS
CREATE TABLE picks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_challenge_id UUID REFERENCES user_challenges(id) ON DELETE CASCADE,
  
  -- Match Info
  match_id VARCHAR(100) NOT NULL,
  match_name VARCHAR(200) NOT NULL,
  match_start_time TIMESTAMP NOT NULL,
  sport VARCHAR(50) DEFAULT 'football',
  league VARCHAR(100),
  
  -- Bet Info
  bet_type VARCHAR(50) NOT NULL, -- 1X2, over_under, btts, asian_handicap, etc.
  selection VARCHAR(100) NOT NULL, -- home, draw, away, over_2.5, etc.
  odds DECIMAL(10,2) NOT NULL,
  
  -- Stake
  stake_amount DECIMAL(10,2) NOT NULL,
  stake_percentage DECIMAL(5,2) NOT NULL,
  
  -- Result
  result VARCHAR(20) DEFAULT 'pending', -- pending, won, lost, void
  profit_loss DECIMAL(10,2) DEFAULT 0,
  profit_loss_raw DECIMAL(10,2), -- avant plafonnement gains
  capped BOOLEAN DEFAULT FALSE,
  capped_amount DECIMAL(10,2),
  
  -- Timestamps
  declared_at TIMESTAMP DEFAULT NOW(),
  settled_at TIMESTAMP,
  
  created_at TIMESTAMP DEFAULT NOW()
);


-- DAILY STATS (pour règles complexes)
CREATE TABLE daily_stats (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_challenge_id UUID REFERENCES user_challenges(id) ON DELETE CASCADE,
  date DATE NOT NULL,
  
  picks_count INTEGER DEFAULT 0,
  daily_profit_loss DECIMAL(10,2) DEFAULT 0,
  daily_gain_raw DECIMAL(10,2) DEFAULT 0,
  daily_gain_capped DECIMAL(10,2) DEFAULT 0,
  cap_applied BOOLEAN DEFAULT FALSE,
  
  balance_at_start DECIMAL(10,2),
  balance_at_end DECIMAL(10,2),
  
  created_at TIMESTAMP DEFAULT NOW(),
  
  UNIQUE(user_challenge_id, date)
);


-- FUNDED ACCOUNTS
CREATE TABLE funded_accounts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  source_challenge_id UUID REFERENCES user_challenges(id),
  
  account_size DECIMAL(10,2) NOT NULL, -- 1000 ou 5000
  current_balance DECIMAL(10,2) NOT NULL,
  status VARCHAR(20) DEFAULT 'active', -- active, suspended, closed
  
  lifetime_payouts DECIMAL(10,2) DEFAULT 0,
  monthly_payout_limit DECIMAL(10,2) DEFAULT 1500,
  lifetime_payout_limit DECIMAL(10,2) DEFAULT 10000,
  
  payout_count INTEGER DEFAULT 0,
  last_payout_at TIMESTAMP,
  
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);


-- PAYOUTS
CREATE TABLE payouts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  funded_account_id UUID REFERENCES funded_accounts(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id),
  
  amount DECIMAL(10,2) NOT NULL,
  status VARCHAR(20) DEFAULT 'pending', -- pending, pending_review, approved, processing, paid, rejected
  
  pix_key VARCHAR(255),
  transaction_id VARCHAR(255),
  
  rejection_reason TEXT,
  
  requested_at TIMESTAMP DEFAULT NOW(),
  reviewed_at TIMESTAMP,
  paid_at TIMESTAMP,
  
  created_at TIMESTAMP DEFAULT NOW()
);


-- PROMO CODES
CREATE TABLE promo_codes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id), -- null si code global

function generateInfluencerCode(influencerName) {
  // Ex: "João Silva" → "JOAO10"
  const clean = influencerName
    .normalize('NFD')
    .replace(/[\u0300-\u036f]/g, '') // enlève accents
    .toUpperCase()
    .replace(/[^A-Z]/g, '');
  const short = clean.substring(0, 6);
  const discount = 10; // ou 15, 20 selon deal
  return `${short}${discount}`;
}

// Middleware Next.js
export function middleware(request) {
  const refCode = request.nextUrl.searchParams.get('ref');
  
  if (refCode) {
    const response = NextResponse.next();
    // Cookie 30 jours
    response.cookies.set('ref_code', refCode, { 
      maxAge: 30 * 24 * 60 * 60,
      httpOnly: false // accessible JS pour tracking
    });
    return response;
  }
  
  return NextResponse.next();
}

async function handlePurchase(userId, amount, challengeType) {
  const refCode = cookies.get('ref_code');
  
  if (refCode) {
    const influencer = await db.influencers.findByCode(refCode);
    
    if (influencer) {
      const commission = calculateCommission(
        amount, 
        influencer.commission_percent,
        influencer.commission_type
      );
      
      await db.influencer_sales.create({
        influencer_id: influencer.id,
        user_id: userId,
        promo_code: refCode,
        sale_amount: amount,
        commission_amount: commission
      });
      
      await db.influencers.update(influencer.id, {
        total_sales: influencer.total_sales + 1,
        total_revenue: influencer.total_revenue + amount,
        total_commission: influencer.total_commission + commission,
        commission_pending: influencer.commission_pending + commission
      });
    }
  }
}


5. Modèles de Rémunération
A. Flat Fee (One-shot)
Influenceur reçoit : €500 fixe
Pour : 1 post Instagram + 3 stories
Livrables : screenshots proof posted
Paiement : 50% avant, 50% après delivery


B. Performance (Commission pure)
Base : €0
Commission : 20% par vente générée
Paiement : mensuel (si > €50 accumulé)


Exemple :
- 30 ventes × €49 = €1,470 CA
- Commission : €1,470 × 20% = €294


C. Hybrid (Recommandé)
Base : €200 garanti
Commission : 10% lifetime sur toutes ventes


Exemple :
- Base : €200
- 25 ventes × €60 moyen × 10% = €150
- Total : €350


Avantage : sécurité influenceur + upside performance


D. Tiered (Gamification)
Niveau 1 (0-10 ventes) : 10% commission
Niveau 2 (11-50 ventes) : 15% commission
Niveau 3 (51-100 ventes) : 20% commission
Niveau 4 (100+ ventes) : 25% commission + bonus

// Cron job : 1er de chaque mois à 9h
async function processInfluencerPayouts() {
  const influencers = await db.influencers.find({
    status: 'active',
    commission_pending: { $gte: 50 }
  });
  
  for (const influencer of influencers) {
    const periodStart = startOfLastMonth();
    const periodEnd = endOfLastMonth();
    
    await db.influencer_payouts.create({
      influencer_id: influencer.id,
      amount: influencer.commission_pending,
      period_start: periodStart,
      period_end: periodEnd,
      status: 'pending',
      pix_key: influencer.pix_key
    });
    
    await db.influencers.update(influencer.id, {
      commission_pending: 0
    });
    
    await sendEmail(influencer.email, 'payout_ready', {
      amount: influencer.commission_pending,
      period: `${formatDate(periodStart)} - ${formatDate(periodEnd)}`
    });
  }
}
